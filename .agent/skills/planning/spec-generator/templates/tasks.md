## 참고사항

- `*` 표시된 작업은 **선택 사항** (MVP에서 건너뛸 수 있음)
- 각 작업은 추적 가능성을 위해 특정 요구사항을 참조합니다
- **체크포인트**에서 검증 후 다음 단계 진행
- **Phase 5 (레거시 제거)** 는 별도 확인 후 진행
- 마이그레이션은 항상 되돌릴 수 있어야 함

## 작성 가이드

### 작업 형식
```markdown
- [ ] {번호}. {작업명} [{FE/BE}]
  - [ ] {번호.번호} {세부 작업} [{필수/선택}]
    - {상세 설명}
    - _요구사항: {번호}, {번호}_
```

### 태그 설명
- `[FE]`: 프론트엔드 작업
- `[BE]`: 백엔드 작업
- `[필수]`: 반드시 해야 하는 작업
- `[선택]` 또는 `*`: 건너뛸 수 있는 작업

### 순서 원칙
1. FE (UI) → 화면 먼저 확정
2. BE (API) → UI 확정 후
3. BE (DB) → 가장 마지막 (되돌리기 어려움)
4. FE (연결) → 마무리
