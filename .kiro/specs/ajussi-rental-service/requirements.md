# Requirements Document

## Introduction

아저씨 렌탈 서비스는 다양한 활동(산책, 대화, 조언 등)을 제공하는 아저씨들과 이러한 서비스를 필요로 하는 사용자들을 연결하는 플랫폼입니다. 사용자는 지역, 태그, 요금 등으로 아저씨를 검색하고, 오픈채팅으로 1차 문의 후 공식 요청을 통해 서비스를 이용할 수 있습니다.

## Requirements

### Requirement 1: 아저씨 목록 및 검색 기능

**User Story:** As a 사용자, I want 아저씨 목록을 보고 검색/필터링할 수 있기를, so that 내 필요에 맞는 아저씨를 쉽게 찾을 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 랜딩페이지에 접속하면 THEN 시스템은 서비스 소개 섹션과 아저씨 카드 리스트를 표시해야 한다
2. WHEN 사용자가 검색/필터를 사용하면 THEN 시스템은 지역, 태그, 요금 기준으로 아저씨 목록을 필터링해야 한다
3. WHEN 아저씨 카드가 표시되면 THEN 시스템은 프로필 사진, 이름/제목, 요금/지역, 태그, 짧은 설명을 포함해야 한다

### Requirement 2: 아저씨 상세 정보 및 요청 기능

**User Story:** As a 사용자, I want 아저씨의 상세 정보를 보고 서비스를 요청할 수 있기를, so that 충분한 정보를 바탕으로 서비스를 신청할 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 아저씨 카드를 클릭하면 THEN 시스템은 상세페이지로 이동하고 프로그레스바를 표시해야 한다
2. WHEN 상세페이지가 로드되면 THEN 시스템은 프로필 사진, 나이, 즐겨찾기, 태그, 제목/설명, 요금, 지역, 주의사항을 표시해야 한다
3. WHEN 사용자가 "오픈채팅 문의" 버튼을 클릭하면 THEN 시스템은 해당 아저씨의 오픈채팅으로 연결해야 한다
4. WHEN 사용자가 "요청하기" 버튼을 클릭하면 THEN 시스템은 요청 모달을 표시해야 한다
5. WHEN 요청 모달이 표시되면 THEN 시스템은 날짜/시간, 소요시간, 장소, 의뢰내용 입력 필드를 제공해야 한다
6. WHEN 사용자가 요청을 완료하면 THEN 시스템은 신청 완료 토스트/팝업을 표시해야 한다

### Requirement 3: 회원가입 및 로그인

**User Story:** As a 사용자, I want SNS 간편 로그인으로 회원가입/로그인할 수 있기를, so that 쉽고 빠르게 서비스를 이용할 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 로그인 페이지에 접속하면 THEN 시스템은 SNS 간편 로그인 버튼들을 표시해야 한다
2. WHEN 사용자가 SNS 로그인을 완료하면 THEN 시스템은 약관 동의 체크박스를 표시해야 한다
3. WHEN 사용자가 약관에 동의하면 THEN 시스템은 회원가입을 완료하고 메인페이지로 이동해야 한다

### Requirement 4: 마이페이지 - 프로필 관리

**User Story:** As a 사용자, I want 내 프로필을 관리할 수 있기를, so that 다른 사용자들에게 정확한 정보를 제공할 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 마이페이지에 접속하면 THEN 시스템은 프로필 이미지, 닉네임, 자기소개 수정 기능을 제공해야 한다
2. WHEN 아저씨 사용자가 프로필을 관리하면 THEN 시스템은 활동 on/off 토글, 소개 제목/설명, 활동 가능 지역, 오픈채팅 URL, 시간당 요금 입력 기능을 추가로 제공해야 한다
3. WHEN 사용자가 프로필 수정을 완료하면 THEN 시스템은 수정 완료 토스트/팝업을 표시해야 한다

### Requirement 5: 의뢰 관리 시스템

**User Story:** As a 사용자, I want 내가 신청한/받은 의뢰를 관리할 수 있기를, so that 서비스 진행 상황을 추적하고 관리할 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 의뢰 관리 페이지에 접속하면 THEN 시스템은 내가 신청한 내역과 내가 받은 내역을 구분하여 표시해야 한다
2. WHEN 의뢰 카드가 표시되면 THEN 시스템은 상대방 이름, 날짜/시간, 장소, 요약, 상태배지를 포함해야 한다
3. WHEN 의뢰 상태가 "대기"이면 THEN 시스템은 신청자에게 [신청취소] 버튼을, 아저씨에게 [수락][거절] 버튼을 제공해야 한다
4. WHEN 의뢰 상태가 "확정"이면 THEN 시스템은 [오픈채팅 연결][성사완료] 버튼을 제공해야 한다
5. WHEN 의뢰 상태가 "완료"이면 THEN 시스템은 [리뷰 남기기] 버튼을 제공해야 한다
6. WHEN 사용자가 취소/완료 버튼을 클릭하면 THEN 시스템은 확인 팝업을 표시해야 한다

### Requirement 6: 아저씨 전용 기능

**User Story:** As a 아저씨, I want 내 활동 통계를 확인하고 관리할 수 있기를, so that 서비스 제공 현황을 파악하고 개선할 수 있다

#### Acceptance Criteria

1. WHEN 아저씨가 마이페이지에 접속하면 THEN 시스템은 이번 달 요청 수, 수락률, 성사 완료 수를 표시해야 한다
2. WHEN 아저씨가 활동 상태를 변경하면 THEN 시스템은 ON/OFF 상태를 즉시 반영해야 한다
3. WHEN 아저씨가 활동 OFF 상태이면 THEN 시스템은 검색 결과에서 해당 아저씨를 제외해야 한다

### Requirement 7: 리뷰 시스템

**User Story:** As a 사용자, I want 서비스 후 리뷰를 남기고 다른 사용자의 리뷰를 볼 수 있기를, so that 서비스 품질을 평가하고 선택에 도움을 받을 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 아저씨 상세페이지에 접속하면 THEN 시스템은 해당 아저씨의 리뷰 목록을 표시해야 한다
2. WHEN 서비스가 완료되면 THEN 시스템은 리뷰 작성 기능을 활성화해야 한다
3. WHEN 사용자가 리뷰를 작성하면 THEN 시스템은 별점과 코멘트를 입력받아야 한다
4. WHEN 아저씨가 마이페이지에 접속하면 THEN 시스템은 내가 받은 리뷰를 표시해야 한다

### Requirement 8: 이용 가이드 및 지원

**User Story:** As a 사용자, I want 서비스 이용 방법과 FAQ를 확인할 수 있기를, so that 서비스를 올바르게 이용하고 문제를 해결할 수 있다

#### Acceptance Criteria

1. WHEN 사용자가 이용 가이드 페이지에 접속하면 THEN 시스템은 이용 절차 안내를 표시해야 한다
2. WHEN 가이드 페이지가 로드되면 THEN 시스템은 주의사항, 안전수칙, FAQ 목록을 아코디언 UI로 제공해야 한다
3. WHEN 사용자가 문의하기를 클릭하면 THEN 시스템은 외부 폼 또는 이메일로 연결해야 한다

### Requirement 9: 시스템 페이지

**User Story:** As a 사용자, I want 오류 상황이나 정책 관련 정보를 확인할 수 있기를, so that 서비스 이용 중 발생하는 문제를 해결하고 정책을 이해할 수 있다

#### Acceptance Criteria

1. WHEN 존재하지 않는 페이지에 접속하면 THEN 시스템은 404 오류 페이지를 표시해야 한다
2. WHEN 사용자가 이용약관/개인정보처리방침 링크를 클릭하면 THEN 시스템은 해당 정책 페이지를 표시해야 한다
3. WHEN 사용자가 신고하기를 요청하면 THEN 시스템은 외부 폼으로 연결해야 한다
4. WHEN 예정된 요청이나 완료된 요청이 있으면 THEN 시스템은 리마인드 배너를 표시해야 한다
